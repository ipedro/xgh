# Migration Guardian

## Problem

Large codebases accumulate deprecated patterns that never get fully migrated. Teams decide on a new architecture (async/await, SwiftUI, DI over singletons) but the old patterns persist indefinitely because:
- No one schedules a migration sprint
- CI doesn't enforce it
- Agents don't know what to migrate unless told

## Idea

A machine-readable config at `.xgh/migrations.yaml` that formalizes "migrate on contact" — whenever an agent touches code matching a deprecated pattern, it migrates it as part of the task. No dedicated migration pass required.

## Config Format

The indexer produces a single pattern catalog at `.xgh/patterns.yaml`. Migrations are derived from it — just entries where `status: deprecated`. One source of truth.

```yaml
# .xgh/patterns.yaml — auto-generated by /xgh-index, reviewed by human

patterns:
  - id: uikit-viewcontroller
    name: "UIViewController-based navigation"
    status: deprecated        # active | gradual | planned | done
    pattern: "UIViewController subclass"
    replace_with: "SwiftUI View"
    replaced_by: swiftui-view
    glob: "Views/**/*.swift"
    priority: medium
    detected_by: xgh-index
    detected_at: 2026-03-16
    files_affected: 12

  - id: completion-handlers
    name: "Callback-based async"
    status: deprecated
    pattern: "completion handler"
    replace_with: "async/await"
    replaced_by: async-await
    glob: "**/*.swift"
    priority: high
    detected_by: xgh-index
    files_affected: 47

  - id: async-await
    name: "Structured concurrency"
    status: stable            # no injection needed

  - id: observation-framework
    name: "@Observable macro"
    status: experimental      # pre | alpha | experimental | stable | deprecated
    pattern: "@Observable"
    glob: "**/*.swift"
    notes: "iOS 17+ only — verify target support before using"

  - id: combine-publishers
    name: "Combine reactive streams"
    status: alpha
    pattern: "Combine publisher chain"
    glob: "**/*.swift"
    notes: "Team evaluating AsyncStream replacement — do not expand usage"

  - id: singleton-pattern
    name: "Singleton via static shared"
    status: deprecated
    pattern: "static let shared"
    replace_with: "dependency injection"
    glob: "**/*.swift"
    message: "Use DI — see .xgh/context-tree/conventions/dependency-injection.md"
    priority: low
```

## Hook Behavior by Status

| Status | Hook injection | Agent action |
|--------|---------------|--------------|
| `deprecated` | "Deprecated pattern found — migrate on contact" | Migrate as part of task |
| `experimental` | "Experimental pattern — verify target support" | Use with caution |
| `alpha` | "Unsettled pattern — do not expand without team sign-off" | Don't add new usage |
| `pre` | "Under evaluation — check team before shipping" | Raise with team |
| `stable` | Silent | Proceed normally |

## Key Principle: Migrate on Contact

Never schedule a migration sprint. Instead:
1. When an agent starts work on a file, `prompt-submit` hook checks `migrations.yaml`
2. If the file matches a pattern, inject into context: *"This file contains deprecated pattern X (migrate → Y, priority: high). Migrate on contact."*
3. Agent migrates as part of the task — opportunistically, not as a separate pass

## Auto-Generation via /xgh-index

The indexer uses multiple signals, ranked by confidence:

### Signal 1: Code comments (highest confidence for intent)

Developers already annotate their intent in comments. Harvest them directly:

| Comment pattern | Inferred status | Confidence |
|----------------|-----------------|------------|
| `DEPRECATED:`, `@deprecated`, `@available(*, deprecated)` | `deprecated` | High |
| `TODO: migrate`, `TODO: replace`, `TODO: remove` | `deprecated` (detected) | Medium |
| `HACK:`, `FIXME:` | `detected` (human reviews) | Low |
| `NOTE: old pattern`, `NOTE: legacy`, `NOTE: workaround` | `detected` | Medium |
| `// remove when X`, `// until X is fixed` | `experimental` or `deprecated` | Medium |

Cluster size amplifies confidence — 15 files with `// TODO: migrate to async/await` → `priority: high`. One file → `priority: low`.

### Signal 2: Pattern coexistence (structural detection)

If the same concern is solved two different ways in the same codebase, the minority pattern is a migration candidate:
- 80% async/await + 20% completion handlers → completion handlers: `detected`
- Both `UIViewController` and SwiftUI views in `Views/` → UIKit: `detected`
- Two different DI approaches → flag both, human decides winner

### Signal 3: Platform/language annotations

Stack-specific deprecation markers:
- Swift: `@available(*, deprecated, renamed:)`
- JS/TS: `/** @deprecated */` JSDoc
- Python: `DeprecationWarning` in source
- Java/Kotlin: `@Deprecated`

These map directly to `status: deprecated` with high confidence.

### Signal 4: Commit messages and PR titles (if git history available)

- `"migrate X to Y"`, `"replace X with Y"`, `"remove legacy X"` → X is deprecated
- Repeated FIXME commits on same files → those files have structural issues worth flagging

### Output

Produces a draft `patterns.yaml` with `source: auto`, `status: detected` for everything. Humans:
- Promote `detected` → `deprecated` / `experimental` / `stable`
- Set `priority`
- Add `goal` and `notes`
- Add `source: human` entries the indexer missed

## Integration Points

- **`/xgh-index`** — generates draft `patterns.yaml` from codebase scan
- **`prompt-submit` hook** — injects relevant pattern warnings when agent touches affected files
- **`convention-guardian` skill** — reads `patterns.yaml` as its enforcement config
- **`/xgh-status`** — shows pattern health (deprecated count, files_affected over time as proxy for migration progress)
- **context tree** — completed migrations archived as decisions in `.xgh/context-tree/architecture/`

## Distinct From convention-guardian

| | convention-guardian | migration-guardian |
|--|--------------------|--------------------|
| Purpose | Enforce current patterns | Track lifecycle of all patterns |
| Config | Implicit (Cipher memory) | Explicit (`patterns.yaml`) |
| Trigger | Any deviation | Proximity to non-stable code |
| Action | Flag + guide | Varies by status (migrate / warn / block) |
| Source | Human-defined | Auto-detected by indexer + human review |

## Stack Agnosticism

Pattern matching is intentionally prose-based, not regex or AST. The glob scopes to the right files; the prose describes the pattern in terms any agent understands without language-specific tooling:

```yaml
# Swift
- pattern: "UIViewController subclass"
  replace_with: "SwiftUI View"
  glob: "Views/**/*.swift"

# JavaScript
- pattern: "nested .then() promise chains"
  replace_with: "async/await"
  glob: "**/*.js"

# Python
- pattern: "os.path.join for path construction"
  replace_with: "pathlib.Path"
  glob: "**/*.py"

# Any stack
- pattern: "God class exceeding 500 lines with mixed responsibilities"
  replace_with: "split by single responsibility principle"
  glob: "**/*"
```

The indexer detects patterns by reading code and docs — same prose, same format, every language.

## Merge Strategy (Human Edits Are Sacred)

`patterns.yaml` is human-owned. The indexer is additive-only and never overwrites human intent.

**Source field signals ownership:**
```yaml
- id: completion-handlers
  source: auto          # indexer owns this entry
  files_affected: 47    # indexer updates this on each run
  status: deprecated    # human promoted this — indexer won't downgrade
  goal: "Q3 migration"  # human added — indexer never touches
```

**Merge rules when `patterns.yaml` already exists:**

| Situation | Indexer action |
|-----------|---------------|
| New pattern detected, not in file | Add with `source: auto`, `status: detected` |
| Entry exists with `source: auto` | Update `files_affected` count only |
| Entry exists with `source: human` | Never touch any field |
| Entry exists, any field has `source: human` annotation | Skip that field |
| `status` set by human (any value) | Never change it |
| `notes`, `priority`, `message`, `goal` present | Never overwrite |
| Entry in file but not detected by scan | Leave it — may be intentional |

**`status: detected`** is the only status the indexer ever writes. Humans promote to `deprecated`, `experimental`, `alpha`, `stable`. This makes intent explicit — the indexer surfaces patterns, humans decide what to do with them.
